## Summary

OpenAI and AWS announced an expanded partnership on April 28, 2026, making OpenAI's frontier models (including GPT-5.5 and GPT-5.4), Codex, and Amazon Bedrock Managed Agents available on AWS — all in limited preview. The launch comes one day after Microsoft's exclusivity arrangement ended, as OpenAI exercises its new right to distribute across all major clouds. Enterprises can now access OpenAI capabilities inside their existing AWS security and compliance infrastructure.

## Key Highlights

- **OpenAI models on Bedrock:** GPT-5.5 and GPT-5.4 available through Bedrock's standard model-access, fine-tuning, and guardrails APIs — with IAM, PrivateLink, and CloudTrail inherited automatically
- **Codex on Bedrock:** OpenAI's coding agent accessible via Codex CLI, desktop app, and VS Code extension inside AWS environments
- **Bedrock Managed Agents (OpenAI-powered):** Production-ready agent deployment with per-agent identity, action logging, and all inference on Bedrock — uses the OpenAI agent harness
- **Timing is notable:** Announced the day after Microsoft's exclusivity on OpenAI distribution ended, and confirmed by Amazon CEO Andy Jassy that OpenAI models are "coming to AWS Bedrock in the coming weeks"
- **Collaboration hosted at an AWS event in San Francisco** with OpenAI CEO Sam Altman and AWS CEO Matt Garman

## Why It Matters

This signals the end of OpenAI's de facto AWS exclusion — enterprise customers who've standardized on AWS can now run GPT-5.5 and agentic Codex workflows without leaving their existing cloud, fundamentally changing how OpenAI's distribution advantages stack up against Anthropic (already on Bedrock) and Google.
